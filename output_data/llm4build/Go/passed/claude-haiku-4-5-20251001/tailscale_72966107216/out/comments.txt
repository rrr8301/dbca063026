On a GitHub Actions runner with a full Ubuntu kernel, `testwrapper` selected 1 package to test (the one failing test was `TestDNSOverTCPIntervalResolver` which timed out in the original CI too). The LLM ran `go test ./...` without `testwrapper`, so all 31 packages ran regardless of kernel capability.
