  [Root cause classification]
  - The failing tests are WebRTC/RTC protocol tests. `TestDataPublishSlowSubscriber` tests that a slow subscriber receiving data channels doesn't cause the SCTP association to crash — the `dtls fatal: conn is closed` WARN from `sctp@v1.9.4/association.go:1041` shows the DTLS connection closed prematurely. `TestNegotiationTiming` tests ICE negotiation timing. Both failures stem from Docker's network stack restrictions: UDP socket buffer sizes are capped (`operation not permitted` when setting `SO_SNDBUF`/`SO_RCVBUF` beyond kernel defaults), and `srflx` ICE candidates (server-reflexive, requiring STUN) can't be gathered inside Docker's NAT without a STUN server reachable from the container's private IP.

  - The 32 fewer tests (478 vs 510) are tests where the ICE connection establishment phase times out waiting for candidate gathering — these tests never finish initialising so they drop out of the count entirely. The LLM correctly used `MallocNanoZone=0` (a macOS env var to avoid false positives — harmless on Linux) and `gotestfmt`, matching the original CI pipeline step, but the WebRTC stack's dependence on unrestricted UDP socket operations is the blocking issue in Docker.
