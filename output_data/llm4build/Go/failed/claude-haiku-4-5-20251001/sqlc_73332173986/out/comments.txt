  [Root cause classification]
  - The 4 MySQL failures all show `Error 1698 (28000): Access denied for user 'root'@'localhost'` — MySQL on Ubuntu installs with the `auth_socket` authentication plugin for the `root` user by default. This means connecting as `root` via TCP (the standard JDBC/Go MySQL driver connection) fails unless the user connects via the Unix socket as the OS `root` user. The sqlc end-to-end examples connect via `root@localhost:3306` over TCP, which the auth_socket plugin rejects.

  - The 1720 fewer tests (49 vs 1769) reflect that the LLM's `gotestsum -- --tags=examples -timeout 20m -failfast ./...` ran only the examples sub-package (49 tests), while the original CI ran the full sqlc test suite including all the unit tests, integration tests, and codegen tests from the root `./...` scope. The original's 2 failures (PostgreSQL-related) were different from the LLM's 4 MySQL failures because the original CI had PostgreSQL and MySQL services configured with proper credentials while the LLM only had a default MySQL install with auth_socket.
