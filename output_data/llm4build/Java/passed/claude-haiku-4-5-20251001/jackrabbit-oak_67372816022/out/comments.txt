  [Root cause classification]
  - MongoDB not started in Docker → 876 DOCUMENT_NS fixture tests fail. The build specifies
    `-Dnsfixtures=SEGMENT_TAR,DOCUMENT_NS`, which runs every integration test twice: once against
    the local segment-tar node store (SEGMENT_TAR) and once against MongoDB (DOCUMENT_NS).
    The original CI (GitHub Actions) has a MongoDB service container available, so all DOCUMENT_NS
    tests can obtain a MongoConnection and pass. In the LLM Docker container no MongoDB process
    is started, so every test that uses the DOCUMENT_NS fixture throws
    `NullPointerException: Couldn't obtain MongoConnection` and fails at JCR repository
    initialization time.
    Because Maven stops at the first failing module (no `--fail-at-end` / `-fae` flag), the build
    aborts after the first module with DOCUMENT_NS failures, causing only 12695 of 24295 total
    tests to execute (roughly half — the tests that had already run in SEGMENT_TAR fixture mode
    in completed modules, plus whatever partial DOCUMENT_NS tests ran before the first failure).
