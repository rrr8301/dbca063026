  [Root cause classification]
  - No Docker socket in Docker → Testcontainers cannot start containers → 4 integration tests
    fail. The `debezium-testing-testcontainers` module's integration tests use Testcontainers
    to spin up Kafka, Apicurio Registry, and MongoDB containers inside the test process.
    Testcontainers requires access to the Docker daemon via `/var/run/docker.sock`. Inside
    Docker, this socket is not mounted, so Testcontainers reports "Previous attempts to find
    a Docker environment failed. Will not retry." and all container-dependent tests fail with
    RuntimeException (Kafka containers) or IllegalStateException (MongoDB replica set).
    In the original CI (GitHub Actions), the runner has Docker installed and the socket is
    accessible, so Testcontainers successfully starts all containers.
    The test count difference (1584 vs 860) is because the LLM runs the `debezium-testing-testcontainers`
    module (which produces 4 failing tests) but the original CI only runs `debezium-connector-mysql`
    tests (with `-pl debezium-connector-mysql -am`). The LLM also runs MySQL with multiple
    profile/version combinations (a matrix of 4 profiles × 3 versions), but many of the
    tests overlap or fail due to profile incompatibilities, resulting in fewer successful test
    runs than the original single MySQL 8.0 run.
