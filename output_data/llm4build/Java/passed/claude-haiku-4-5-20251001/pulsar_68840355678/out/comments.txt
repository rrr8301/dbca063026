  - Pre-built Pulsar Docker image not available → SHADE_RUN integration tests fail with
    IllegalStateException. The original CI is a two-phase process: a prior CI job builds Pulsar
    and packages it as `java-test-image.tar.gz` which is downloaded as a CI artifact and loaded
    into the local Docker daemon (`docker load -i /tmp/java-test-image.tar.gz`). The SHADE_RUN
    integration tests (`SmokeTest`, `SimpleProducerConsumerTest`) start a Pulsar cluster from
    this pre-built image and test the shaded clients against it.
    The LLM attempts to replicate this flow but does not have the pre-built image or the
    pre-built Gradle artifacts. The LLM's `./gradlew assemble -x test` builds from scratch
    (which succeeds), but when SHADE_RUN runs the integration tests, the Pulsar cluster cannot
    start because the Docker image is not available (and even if Docker were available inside
    the container, the `java-test-image.tar.gz` was never loaded). All 34 integration tests
    fail at setup with `IllegalStateException`, and 34 more tests are skipped (the `stateCheck`
    phases that depend on a running cluster).
    The original CI's 0 tests reported is because the test results from the pre-built SHADE_RUN
    integration tests are reported via a custom action (`apache/pulsar-test-infra/action-junit-report`)
    rather than standard Gradle XML format, making the stats extractor report 0.
