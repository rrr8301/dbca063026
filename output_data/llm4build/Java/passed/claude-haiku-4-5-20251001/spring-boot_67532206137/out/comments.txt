  [Root cause classification]
  - Java version too old → build fails after 1177 tasks. The Spring Boot project enforces a
    minimum Java version of 25. The LLM Dockerfile installs OpenJDK 21 (Ubuntu package), and
    when `./gradlew build` executes the Java version enforcer after compiling 1177 tasks, it
    throws `This project should be built with Java 25 or above`, aborting the build before the
    remaining ~4720 tasks (and all test tasks) can run.
    The original CI builds with Java 21 via `actions/setup-java` (Temurin JDK 21) and uses the
    project's custom `.github/actions/prepare-gradle-build` action which configures Spring
    Boot-specific Gradle properties that bypass or satisfy the version check for the CI run
    (possibly using a Gradle toolchain that downloads Java 25 separately, or bypassing the
    enforcer via a CI-specific property). Additionally, the original CI uses the Develocity
    remote build cache (GRADLE_BUILD_ACTION_CACHE_RESTORED: true), so many tasks are loaded
    FROM-CACHE rather than executed, which is why the original reports 5897 "executed" tasks
    (tasks include both executed and cache-restored tasks in Gradle's count). The LLM has no
    access to this cache and builds everything from scratch, but fails mid-way.
