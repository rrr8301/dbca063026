  - Two combined issues cause the test and count differences:
    (1) Different test catalog → LLM runs 17,290 more tests. The original CI downloads a
    `combined-test-catalog.txt` CI artifact (sha256:899d12de...) that specifies a curated subset

    a different (larger) set of 64,302 tests. The combined catalog is built during the CI pipeline
    and may exclude certain tests (e.g., known flaky or slow tests), while the local catalog
    includes all registered tests.
    (2) Docker runs as root → file/directory permission tests fail. Several failing tests call
    `setWritable(false)` / `chmod 000` on a temp directory and then expect the production code
    to throw an exception when trying to write to that directory. Running as root bypasses all
    filesystem permission checks, so the writes succeed silently instead of throwing, causing
    `assertThrows()` to fail (nothing was thrown). This explains the 78 failures in the LLM
    vs 6 in the original CI: some original 6 failures are pre-existing known failures (counted
    with `--continue`), while the LLM adds root-related permission bypass failures on top.
    `GlobalStateManagerImplTest.shouldLogWarningMessageWhenIOExceptionInCheckPoint()` fails
    because the test makes a path non-writable to force an IOException log message, but root
    user writes successfully without triggering the warning.
