    obtains the timezone display name from the JDK's locale data (e.g., "hora estándar de Palmer"),

    On the original CI (Eclipse Temurin JDK 25.0.2), the CLDR locale data for `es_US` provides
    timezone display names that `FastDateParser`'s `TimeZoneStrategy` cannot reverse-parse —
    the displayed string "hora estándar de Palmer" is not in its timezone name lookup table for
    that locale. On the LLM Docker container (Ubuntu OpenJDK 25.0.2), the CLDR data bundled with
    the ubuntu `openjdk-25-jdk` package either uses different Spanish timezone abbreviations that
    FastDateParser can match, or the JVM locale data reports these timezones as unavailable and
    the test skips them. The result is that 7 parametrized test instances fail in the original CI
    but pass (or are skipped) in the LLM environment — an inverted failure pattern caused solely
    by JDK vendor/distribution differences in bundled locale/CLDR data.
