    Amazon Redshift cluster (`setup-aws-redshift.sh` creates an ephemeral Redshift cluster).
    The 13 tests are the `-P fte-tests` test suite which connects to a real Redshift instance.
    `testCreateTable` in `TestRedshiftQueryFailureRecoverySmokeTest` fails because the test
    simulates query failure recovery scenarios against a live Redshift cluster, and a specific
    query failure recovery path doesn't work as expected in the CI's Redshift version.
    The LLM's run.sh has the cloud Redshift test block gated on `AWS_ACCESS_KEY_ID` being set:
    ```
    if [ -n "${AWS_ACCESS_KEY_ID:-}" ] && [ -n "${REDSHIFT_SUBNET_GROUP_NAME:-}" ]; then
    ```
    Since Docker has no AWS credentials, the cloud Redshift tests are skipped entirely.
    The LLM runs only the 4 non-cloud unit tests from `mvn test -pl plugin/trino-redshift`
    (without the `-P fte-tests` profile), which all pass. The 9 missing tests (13 - 4) are
    the cloud-dependent `fte-tests` profile tests that require a live Redshift cluster.
