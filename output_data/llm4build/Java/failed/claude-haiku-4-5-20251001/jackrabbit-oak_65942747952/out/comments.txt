  [Root cause classification]
  - MongoDB is not running in the Docker container: `-Dnsfixtures=SEGMENT_TAR,DOCUMENT_NS`
    instructs the JCR TCK and integration tests to run each test against both the
    SEGMENT_TAR (file-based) and DOCUMENT_NS (MongoDB-backed) NodeStore implementations.
    The Dockerfile installs Maven and JDK but does not install or start MongoDB. When
    `OakMongoNSRepositoryStub` is constructed (the stub that backs the DOCUMENT_NS fixture),
    it calls `MongoUtils.getConnection()` which returns null because no MongoDB server is
    listening on localhost:27017. The null connection causes `NullPointerException: Couldn't
    obtain MongoConnection` at `OakMongoNSRepositoryStub.java:55`, which is wrapped in a
    `RepositoryException` and thrown from the constructor. Every JCR TCK test class that uses
    the DOCUMENT_NS fixture fails in its test setup with this exception, producing 876 errors.
    The 10438 fewer tests in the LLM (12707 vs 23145) are the DOCUMENT_NS variants of those
    same test classes that should have run but errored in setUp before counting. The original
    CI's GitHub Actions runner has MongoDB available (either pre-installed on ubuntu-latest or
    provisioned by the workflow), so all DOCUMENT_NS tests pass and only the OSGiIT tests
    (which test the OSGi deployment bundle wiring) fail with 6 errors.
