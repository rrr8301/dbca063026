  [Root cause classification]
  - Pulsar's integration tests use Testcontainers to spin up a full Pulsar cluster (broker,
    bookie, ZooKeeper) as Docker containers inside the test environment. Testcontainers
    requires `/var/run/docker.sock` to be available and accessible. The LLM's Docker
    container does not mount the host's Docker socket (no `-v /var/run/docker.sock:/var/run/docker.sock`
    in the docker run command), so Testcontainers throws `IllegalStateException: Could not
    find a valid Docker environment` in every test class's `setupBeforeClass`. All 35 test
    class-level setUp failures cascade to all their test methods being counted as failures.
    The original CI's GitHub Actions runner has Docker available as a service and mounts
    the Docker socket to the workflow environment, so Testcontainers can launch containers.
    The original CI's 1 failure (`testTopicReplication[non-persistent]` in GeoReplicationTest)
    is an unrelated flakiness issue in the geo-replication test logic, not a Docker access problem.
    The test count difference (92 vs 58) occurs because `run_integration_group.sh MESSAGING`
    runs a broader set of test classes than the original CI's `-DintegrationTestSuiteFile=pulsar-messaging.xml`
    filter, so more test classes are attempted and all fail in setUp.
