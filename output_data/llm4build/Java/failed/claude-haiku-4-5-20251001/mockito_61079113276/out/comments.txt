  [Root cause classification]
  - The GitHub Actions runner for this job installs multiple JDK versions (11, 17, 21, 25)
    via `actions/setup-java`, making all JDK toolchains available for Gradle. Even though
    `-Pmockito.test.java=11` sets the primary test JVM to JDK 11, Gradle's toolchain API
    can still resolve the JDK 21 toolchain for `java-21-tests` integration tests, allowing
    those modules to run and contribute to the 2556 total. The Docker container uses
    `eclipse-temurin:17-jdk-jammy` as base and only installs `openjdk-11-jdk` via apt;
    no JDK 21 is present. When Gradle resolves toolchains for `java-21-tests`, it cannot
    find a JDK 21 installation and SKIPS all test tasks in that module
    (`testWithBytebuddyAgent`, `testWithEnableDynamicAgentLoading`, `testWithMockitoAgent`,
    `test`). Similarly, other modules that require JDK 21+ are SKIPPED, reducing the
    count from 2556 to 290. The 1 Gradle failure and 1 mocha failure in both environments
    are the same `inline-mocks-tests` test, which fails for an unrelated reason in both.
    The mocha count difference (154 vs 30) mirrors the same pattern: Node.js test suites
    associated with the skipped Java modules also don't run.
