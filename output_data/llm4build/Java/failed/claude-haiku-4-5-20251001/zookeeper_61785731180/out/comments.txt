  [Root cause classification]
  - Two distinct Docker environment issues cause the 8 failures:
    (1) Docker runs as root → read-only directory tests fail: `testReadOnlySnapshotDir`,
    `testReadOnlyTxnLogDir`, and `testUnreadableFileInput` set directories or files to
    read-only via `chmod 000` / `setWritable(false)` and then expect ZooKeeper to either
    fail to start (snapshot/txnlog tests: `expected: <false> but was: <true>`) or fail to
    read a file. Root ignores all permission bits, so the server starts successfully and
    file reads succeed, causing all three tests to fail their assertions.
    (2) No IPv6 in Docker → IPv6 quorum tests time out: `testQuorumV6`,
    `shouldStartClusterWithMultipleAddresses_IPv6`, `shouldStartClusterWhenSomeAddressesAreUnreachable_IPv6`,
    and `shouldReconfigIncrementally_IPv6` start ZooKeeper quorum peers bound to `::1`
    (IPv6 loopback). Docker's network stack doesn't expose IPv6 networking by default,
    so the servers cannot bind to IPv6 addresses, fail to come up, and the tests time out
    after 30 seconds. `testClientServerUnifiedPortWithCnxnClassName` likely also fails
    due to an SSL/IPv6 binding issue (15-second timeout). The original CI's 1 failure
    (`testRequestThrottler`) is a timing-sensitive throttling test that is flaky under
    GitHub Actions load but passes in Docker's single-threaded environment.
