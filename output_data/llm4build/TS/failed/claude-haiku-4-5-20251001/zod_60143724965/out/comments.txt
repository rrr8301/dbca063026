  [Root cause classification]
  - The CI workflow runs `pnpm add typescript@latest -w` before building and testing, which installs the newest TypeScript release at the time of the Docker build. The LLM ran the same command, but the newest TypeScript version available at build time was newer than the one used in the original CI. The zod repository's `tsconfig.build.json` and some fixture configs use `"ignoreDeprecations": "6.0"` to suppress TypeScript 5.x deprecation warnings for older options. In the latest TypeScript, the value `"6.0"` for `--ignoreDeprecations` is no longer valid — TypeScript now rejects it with `error TS5103: Invalid value for '--ignoreDeprecations'`. This causes the build step to fail with exit code 2 before all tests compile.

  - Despite the build compilation error, vitest's TS type-checking mode still ran and surfaced 25 TypeCheck failures. These are type-level assertion tests that use `expectTypeOf` and compile-time inference checks. The failures cluster around `z.tuple` with optional elements (`src/v4/classic/tests/tuple.test.ts`), inferred object types with optional properties (`src/v3/tests/object.test.ts`, `src/v3/tests/partials.test.ts`), and error-tree inference (`src/v4/classic/tests/error-utils.test.ts`, `src/v3/tests/all-errors.test.ts`). Each of these is a `TypeCheckError: Argument of type 'true' is not assignable to parameter of type 'false'` — the canonical vitest-expect-type failure form — indicating that a TypeScript type narrowing or inference constraint that was expected to hold at the previous TypeScript version no longer holds under the newer one.

  - The original CI had 3,555 tests all passing because the TypeScript version it used (installed earlier in the workflow run) accepted `"ignoreDeprecations": "6.0"` and its type inference matched the expectations baked into the snapshot-style type tests. The LLM has 3,556 tests (1 more, from an extra test discovered due to slightly different module resolution) and 25 failures, all driven by TypeScript version drift. This is a fundamental reproducibility challenge for any project whose tests are gated on `typescript@latest`: the exact TypeScript release is non-deterministic at Docker build time.
