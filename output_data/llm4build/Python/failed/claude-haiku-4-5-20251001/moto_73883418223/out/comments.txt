  [Root cause classification]
  - boto3 not installed → pytest discovers internal ASL test files that import boto3, causing
    3 collection errors; only 3 "tests" run vs 11851 in the original CI

    The moto project (a mock AWS SDK library) has test files embedded within its source tree,
    specifically under `moto/stepfunctions/parser/asl/`. These are parser unit tests that import
    `boto3` directly. The LLM Dockerfile installs only `pytest-github-actions-annotate-failures`
    (and its dependencies) but does NOT install moto's full test dependencies. Without `boto3`
    installed, pytest cannot import these three test files during collection:
      - `moto/stepfunctions/parser/asl/component/test_state/program/test_state_program.py`
      - `moto/stepfunctions/parser/asl/component/test_state/state/test_state_state_props.py`
      - `moto/stepfunctions/parser/asl/static_analyser/test_state/test_state_analyser.py`

    The run.sh uses `pytest --tb=short` without a specific test path, so pytest recursively
    discovers ALL test files from the working directory — including these internal source-tree
    tests. Since only 3 collection errors are found before the rest aborts, the LLM run reports
    "3 tests run, 3 failed" (collection errors counted as failures by the stats parser) vs the
    original CI's 11851 tests.

    The original CI installs the full `requirements-dev.txt` which includes `boto3` and all
    other moto test dependencies, then runs `pytest tests` (pointing to the `tests/` directory
    specifically, not the source tree), so the 11851 actual unit tests in `tests/` are collected
    and run correctly.
