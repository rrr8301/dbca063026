  [Root cause classification]
  LLM run.sh passes `--reverse` to .ci/test.sh, which causes additional test categories
  (including oss-fuzz tests) to run — 3784 tests vs 1419 in original

  The original CI's `.ci/test.sh` script checks an environment variable `$REVERSE`. When
  `$REVERSE` is unset (original CI), the script runs pytest with the standard test selection
  and pytest-reverse is not installed. When `.ci/test.sh --reverse` is passed (LLM Docker),
  the script installs `pytest-reverse` and changes the test collection behavior. In addition
  to reversing test execution order, the `--reverse` invocation also alters the test scope:
  the LLM run collects and executes `Tests/oss-fuzz/test_fuzzers.py::test_fuzz_images` tests
  (which test against each image file in `Tests/images/` — hundreds of parametrized variants),
  while the original CI's `.ci/test.sh` (without `--reverse`) uses `--ignore=Tests/oss-fuzz`
  or equivalent filtering to exclude these fuzzing tests.

  This results in 3784 total tests in LLM (1419 standard tests + 2365 fuzz image tests from
  `Tests/oss-fuzz/test_fuzzers.py`) vs 1419 in the original CI. Additionally, 15 fewer tests
  are skipped in LLM (16 vs 31) because test ordering with `pytest-reverse` changes how some
  fixture-conditional skips are evaluated.

  The actual test FAILURE is the same in both environments:
  `test_image_save` fails with `AttributeError: 'Image' object has no attribute 'encoderinfo'`
  when saving a CMYK image to PNG format. This is a pre-existing failure present in both.
  FR=false is driven entirely by the test count discrepancy from the `--reverse` flag.
