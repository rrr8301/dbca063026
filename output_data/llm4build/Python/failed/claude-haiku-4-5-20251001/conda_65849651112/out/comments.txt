  [Root cause classification]
  Two issues cause the 52 additional test failures:

  (1) conda PluginError: `signature-verification` post_solve hook registered twice
    The conda development build (installed from `/workspace/` via `python -m conda init --all`)
    registers the `conda_post_solves` hook for signature verification from the dev codebase.
    The Miniconda base environment (pre-installed in the Docker image at `/opt/miniconda/`) also
    has conda's `signature-verification` post_solve plugin registered. When the test process
    uses both the dev conda (from `/workspace/`) and the system conda from miniconda in the same
    Python process, two identical `CondaPostSolve(name='signature-verification')` hooks appear
    in the plugin manager simultaneously.

    The plugin manager's `get_hook_results("post_solves")` detects duplicate names and raises:
      `PluginError: Conflicting plugins found for 'post_solves':
       CondaPostSolve(name='signature-verification', ...)
       CondaPostSolve(name='signature-verification', ...)`
    This error fires whenever any integration test triggers a `conda install`, `conda create`,
    or `conda update` operation — which covers most of the 52 failing tests (cli, env, shell
    integration tests all involve conda solve operations).

    The original CI runs in a clean GitHub Actions runner where only the test-configured conda
    is active, not a pre-existing miniconda installation. No duplicate plugins exist there.

  (2) PowerShell not installed → test_shell_available[powershell] errors at collection
    `tests/shell/test_powershell.py::test_shell_available[powershell]` raises a
    `FileNotFoundError: Shell(name=('powershell', 'pwsh', 'pwsh-preview'), path=None, exe=None)
    not found`. PowerShell (`pwsh`) is not installed in the ubuntu:24.04 Docker container,
    but is pre-installed on the GitHub Actions ubuntu-latest runner.
