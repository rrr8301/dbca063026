  not yet defined in peft.utils.integrations → 46 new ImportError failures

  The PEFT workspace source (`peft==0.18.2.dev0`) has `src/peft/utils/save_and_load.py:459`
  which contains:
    `from peft.utils.integrations import convert_peft_adapter_state_dict_for_transformers`
  This import was added to handle transformers v5 state dict format conversion, but the
  corresponding function `convert_peft_adapter_state_dict_for_transformers` was NOT added to
  `/workspace/src/peft/utils/integrations.py` in this source snapshot.

  The import is a lazy import inside the `save_pretrained()` function path, triggered only for
  specific model types (IA3Config, LoRA with device maps, etc.) that call `save_pretrained()`.
  Additionally, `test_helpers.py::TestScalingAdapters::test_transformers_pipeline` fails with
  `ImportError: cannot import name '_maybe_shard_state_dict_for_tp' from 'peft.utils.save_and_load'`
  — a second missing symbol in save_and_load.py related to tensor-parallel sharding support that
  was also referenced before being defined in the workspace.

  The original CI ran earlier when `save_and_load.py` had NOT yet added these transformers v5
  conversion imports, so those 46 tests didn't encounter the ImportError. By the time the LLM
  Docker was built and run, the workspace code added the references to
  `convert_peft_adapter_state_dict_for_transformers` and `_maybe_shard_state_dict_for_tp` in
  `save_and_load.py` without simultaneously adding the functions to `integrations.py`, leaving
  the codebase in a partially-committed state.
