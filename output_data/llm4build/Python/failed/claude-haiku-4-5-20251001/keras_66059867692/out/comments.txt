  [Root cause classification]
  JAX 0.10.0 API change breaks `jax_custom_fit_test.py` → `set -e` aborts before `pytest keras` runs

  The LLM's `run.sh` begins with `set -e` (exit on error). The sequence of pre-test
  integration scripts runs in order:
    1. `python integration_tests/import_test.py`  → passes
    2. `python integration_tests/numerical_test.py` → passes
    3. `python integration_tests/jax_custom_fit_test.py` → FAILS with TypeError

  At step 3, the script raises:
    `TypeError: test_custom_fit.<locals>.CustomModel.train_step() missing 1 required
     positional argument: 'data'`

  Because `set -e` is active, the Python script failure (non-zero exit code) causes the
  entire run.sh to abort immediately. The main test command `pytest keras ...` is never
  executed. Only 2 tests are captured in the output — these come from an earlier pytest
  invocation by the LLM agent during its build phase (`python -m pytest integration_tests/
  basic_full_flow.py`), not from the keras unit test suite.

  The TypeError root cause is a JAX version API change:
    - Original CI installed: jax==0.9.1, jaxlib==0.9.1, keras==3.13.2
    - LLM Docker installed: jax==0.10.0, jaxlib==0.10.0 (newer version from pip)

  In JAX 0.10.0, the internal training loop in Keras's JAX backend changed how
  `train_step(self, data)` is called. The `jax_custom_fit_test.py` script defines a
  `CustomModel` class that overrides `train_step`. In the JAX 0.10.0 / Keras environment,
  the custom `train_step` is invoked without the `data` argument (the caller no longer
  passes `data` positionally), causing:
    `TypeError: train_step() missing 1 required positional argument: 'data'`

  In the original CI with jax==0.9.1, the `train_step(data)` calling convention matched
  the expected signature, so the script succeeded and `pytest keras` ran to completion,
  reporting 13457 tests (1 failure: `MultiHeadAttentionTest::test_lora`).
