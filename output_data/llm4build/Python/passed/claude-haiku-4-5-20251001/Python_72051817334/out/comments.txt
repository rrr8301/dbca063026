  [Root cause classification]
  Two missing system libraries cause collection errors that abort pytest entirely:

  (1) Missing libGL.so.1 (OpenGL shared library) — 21 collection errors
    The Python algorithms repository (TheAlgorithms/Python) includes computer vision and
    image processing algorithms that import OpenCV (`import cv2`). OpenCV on Linux requires
    `libGL.so.1` (part of the OpenGL Mesa library, package `libgl1-mesa-glx` or `libgl1`).
    The LLM Dockerfile (`ubuntu:latest`) installs only `libhdf5-dev` as a system library
    but not `libgl1-mesa-glx`. At collection time, pytest tries to import 21 modules
    (computer_vision/*.py, digital_image_processing/*.py, data_compression/
    peak_signal_to_noise_ratio.py) which import cv2, triggering:
      `ImportError: libGL.so.1: cannot open shared object file: No such file or directory`
    These 21 errors count as collection errors in the summary.json (num_tests_failed=23).

  (2) Missing tkinter — 2 collection errors
    Two fractal modules (fractals/sierpinski_triangle.py, fractals/vicsek.py) import
    `tkinter` for visualization. `ubuntu:latest` with Python from the deadsnakes PPA does
    not include the `python3-tk` package, so `import tkinter` raises:
      `ModuleNotFoundError: No module named 'tkinter'`

  Pytest aborts with "Interrupted: 23 errors during collection" after encountering the
  collection errors, and 0 tests actually run (the 23 in summary.json are all collection
  errors, not test failures from running test functions). The original CI (ubuntu-latest
  GitHub Actions runner) has `libgl1-mesa-glx` and `python3-tk` pre-installed.
