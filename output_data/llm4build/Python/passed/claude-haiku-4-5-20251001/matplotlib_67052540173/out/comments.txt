  [Root cause classification]
  Two combined effects:

  (1) setuptools-scm version scheme rename → 828 DeprecationWarning failures
    The LLM Docker container installs a newer version of `setuptools_scm` (or its
    `vcs_versioning` dependency) that renamed the version scheme `release-branch-semver`
    to `semver-pep440-release-branch`. matplotlib's `pyproject.toml` specifies
    `version_scheme = "release-branch-semver"` in `[tool.setuptools_scm]`. When any
    test calls `setuptools_scm.get_version()` (e.g., to embed the version string in
    a saved figure), the newer setuptools-scm raises:
    `DeprecationWarning: Version scheme 'release-branch-semver' has been renamed to
     'semver-pep440-release-branch'. Please update your configuration.`
    matplotlib's test suite converts `DeprecationWarning` into test errors (via
    `filterwarnings = ["error::DeprecationWarning"]` in `pyproject.toml`), so
    every test that calls into the version-lookup code path fails with this warning.
    The original CI has a pinned/older setuptools-scm version that uses the old scheme
    name without raising a deprecation warning.

  (2) --maxfail=50 with 44 parallel workers stops run after 50+ failures
    pytest is invoked with `--maxfail=50`. With 44 pytest-xdist workers running
    concurrently, once 50 failures accumulate across all workers, xdist sends a
    stop signal: `Interrupted: stopping after 50 failures`. However, because 44
    workers run in parallel, tests already in-flight on each worker continue to
    complete, resulting in 803 total failures (+ 25 errors = 828 combined) before
    the run actually stops. The remaining ~6830 tests (out of 10333 collected) are
    never scheduled, leaving the LLM run with only 3503 tests executed vs 10335 in
    the original CI (which had 0 failures and ran all 10335 tests to completion).
