  [Root cause classification]
  Two combined issues:

  (1) Missing WebDriver binaries: chromedriver and geckodriver not installed
    `test_browser_smoke[Chrome]` and `test_browser_smoke[Firefox]` use Selenium to launch
    real browser instances (Chrome and Firefox respectively) to smoke-test the Dash testing
    browser utilities. The tests require `chromedriver` (for Chrome) and `geckodriver`
    (for Firefox) executables in PATH. The LLM Dockerfile (ubuntu:24.04) installs nodejs,
    npm, and Python build tools but not Chromium/Chrome or Firefox with their WebDriver
    binaries. All 20 webdriver initialization retries fail with:
      `FileNotFoundError: [Errno 2] No such file or directory: 'chromedriver'`
    and `WebDriverException: 'chromedriver' executable needs to be in PATH`.
    The GitHub Actions ubuntu-latest runner has Chromium/Chrome pre-installed with chromedriver.

  (2) jest tests absent from LLM run (79 tests missing entirely)
    The original CI runs `npm ci` (clean install from `package-lock.json`) before `npm run
    citest.unit`. The `npm run citest.unit` script runs both jest (for JavaScript components)
    and pytest (for Python). The LLM run.sh uses `npm run build` instead of `npm ci`. Without
    a proper `npm ci` step, the jest test runner and its dependencies (configured via
    `package-lock.json`) are not installed. When `npm run citest.unit` executes, the jest
    portion silently skips or fails to collect, resulting in 0 jest tests run vs 79 in
    the original CI.
