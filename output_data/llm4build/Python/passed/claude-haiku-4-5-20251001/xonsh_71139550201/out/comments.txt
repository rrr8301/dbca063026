  [Root cause classification]
  Two combined effects of running on Linux Docker instead of macOS-latest CI:

  (1) Bash completion tests: @skip_if_on_darwin tests run on Linux but fail (no bash-completion)
    19 tests in test_bash_completer.py are decorated with `@skip_if_on_darwin`, meaning they
    are SKIPPED on macOS (original CI) but RUN on Linux (Docker). This explains why the LLM
    runs 1067 more tests (7169 vs 6102). On Linux Docker (python:3.13-slim), the `bash-completion`
    package is not installed. The `complete_from_bash()` function invokes bash's completion
    mechanism (e.g., `compgen -W` or sourcing bash completion scripts) to get completions for
    xonsh commands like `xonsh --d`, paths, and shell globs. Without `bash-completion` configured
    in the container, bash returns no completions, so `bash_completions == set()` instead of
    the expected sets like `{'--debug', '--debugger'}` or `{'/proc/'}`.

  (2) Signal handling tests: xonsh SIGHUP behavior differs between macOS and Linux
    `test_catching_exit_signal` sends SIGHUP to xonsh and expects exit code > 0.
    On macOS, xonsh exits with non-zero on SIGHUP; on Linux, it exits with 0 (ret == 0).
    `test_forwarding_sighup` and `test_on_postcommand_waiting` time out after 5 seconds —
    signal propagation and process group management differ between macOS (kqueue) and Linux
    (epoll) in ways that affect xonsh's interactive process handling.
