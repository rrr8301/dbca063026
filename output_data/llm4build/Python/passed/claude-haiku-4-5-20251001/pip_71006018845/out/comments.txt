  [Root cause classification]
  Two separate issues:

  (1) Port conflict in test_download_metadata_server: [Errno 98] Address already in use
    The functional tests for `test_download_metadata_server` start a local HTTP server using
    Python's `socketserver` module. With pytest-xdist running ~80 parallel workers, two
    worker processes attempt to bind a server socket to the same port simultaneously.
    The second bind attempt fails with `OSError: [Errno 98] Address already in use`. This
    is a test isolation issue caused by running the functional server-based tests under
    heavy parallelism. The original CI may have run with fewer workers or with a different
    port assignment strategy that avoided collisions.

  (2) test_pip_wheel_readonly_cache: Docker root user bypasses chmod 000
    The test `test_pip_wheel_readonly_cache` verifies that when pip's cache directory is
    set to a read-only path, pip prints "The cache has been disabled." and does not write
    to the cache. The test marks the cache directory as read-only using `chmod 000`.
    However, in the Docker container, the process runs as root. On Linux, root bypasses
    file permission checks, so `chmod 000` does NOT prevent root from reading or writing
    to the directory. Pip successfully writes the built wheel to the "read-only" cache
    (stored in `.../cache/wheels/4b/fb/02/...`) without printing the expected disable
    message. The assertion `assert 'The cache has been disabled.' in ...` fails because
    pip never detected that the cache was inaccessible.
