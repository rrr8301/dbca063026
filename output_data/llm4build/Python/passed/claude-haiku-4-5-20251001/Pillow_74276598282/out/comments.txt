  [Root cause classification]
  Two combined effects:

  (1) Missing clipboard utility (xclip/wl-paste) -- test_grabclipboard_wl_clipboard_errors fails
    `TestImageGrab::test_grabclipboard_wl_clipboard_errors[--clear]` tests the error-handling
    path when a Wayland clipboard tool is invoked with the `--clear` flag. The test requires
    either `wl-paste` (Wayland clipboard) or `xclip` (X11 clipboard) to be installed.
    The LLM Dockerfile installs Wayland/X11 development libraries (libwayland-dev, libxcb1-dev)
    but does NOT install the `xclip` or `wl-paste` clipboard binaries. In the Pillow source,
    `ImageGrab.grabclipboard()` checks:
      `elif shutil.which("xclip") and session_type in ("x11", None):`
    Since neither `xclip` nor `wl-paste` are on PATH, Pillow raises:
    `NotImplementedError: wl-paste or xclip is required for ImageGrab.grabclipboard() on Linux`
    On the GitHub Actions runner (ubuntu-latest), `xclip` is pre-installed and the test
    proceeds to test the `--clear` error path, which passes correctly.

  (2) pytest stopped at first failure (5274 - 566 = 4708 tests not run)
    The `.ci/test.sh` script invokes pytest with the `-x` (stop-at-first-failure) flag.
    Additionally, the run.sh sets `REVERSE=--reverse` (pytest-reverse plugin), which runs
    tests in reversed collection order vs normal alphabetical order. With reversed ordering,
    `Tests/test_imagegrab.py` runs early in the sequence (around the 10% mark). When this
    test fails, pytest immediately stops the run with `stopping after 1 failures`. As a
    result, only 566 tests execute (roughly the first 10% of the full suite in reversed order),
    while the original CI completes all 5274 tests. Both builds collected the same 5272 items,
    but the failure aborts the LLM run early.
