#7 16.27 libheif makes use of libde265 for HEIF image decoding and x265 for encoding. For AVIF, libaom, dav1d, or rav1e are used as codecs.
#7 16.27 
#7 16.27 libheif has support for decoding
#7 16.27 - tiled images
#7 16.27 - alpha channels
#7 16.27 - thumbnails
#7 16.27 - reading EXIF and XMP metadata
#7 16.27 - reading the depth channel
#7 16.27 - multiple images in a file
#7 16.27 - image transformations (crop, mirror, rotate)
#7 16.27 - overlay images
#7 16.27 - plugin interface to add alternative codecs for additional formats (AVC, JPEG)
#7 16.27 - decoding of files while downloading (e.g. extract image size before file has been completely downloaded)
#7 16.27 - reading color profiles
#7 16.27 - heix images (10 and 12 bit, chroma 4:2:2)
#7 16.27 
#7 16.27 The encoder supports:
#7 16.27 - lossy compression with adjustable quality
#7 16.27 - lossless compression
#7 16.27 - alpha channels
#7 16.27 - thumbnails
#7 16.27 - save multiple images to a file
#7 16.27 - save EXIF and XMP metadata
#7 16.27 - writing color profiles
#7 16.27 - 10 and 12 bit images
#7 16.27 - monochrome images
#7 16.27 
#7 16.27 See https://github.com/strukturag/libheif for further information.
