  [Root cause classification]
  - The LLM ran `cargo test --all-features --no-fail-fast` instead of `cargo nextest run`. The `--all-features` flag enables the `uv-keyring` crate, which compiles in OS-level secret-service (D-Bus/GNOME Keyring) integration for credential storage. uv's auth integration tests spin up a local mock HTTP registry (`127.0.0.1:<random-port>`) with basic-auth, and after each test they attempt to remove the stored credential from the OS keyring. In the headless Docker container there is no D-Bus session or secret-service provider, causing every credential-cleanup call to fail with "no secret service provider or dbus session found". This cascades into 2557 test failures.

  - The original CI used `cargo nextest run` (without `--all-features`), which does not enable the `uv-keyring` feature and therefore does not invoke the OS keyring at all — all 3705 tests pass. Additionally, `cargo nextest` runs each test in a separate subprocess, providing stronger isolation between the mock HTTP servers so they do not share port state. The LLM's `cargo test` runs tests in threads within the same process, further increasing the chance of port collisions in the mock-server-heavy auth and build test suites.
