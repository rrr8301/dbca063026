  [Root cause classification]
  Two separate failure causes:

  (1) Original CI only: insta snapshot regressions in auth and secret_service tests
    The uv `auth` integration tests verify authentication flows (native auth tokens,
    host fallback, prefix matching) and `secret_service` tests verify credential storage
    behavior. These tests use `insta` for snapshot assertions. The snapshots at their
    respective locations are outdated — the current implementation produces different output
    than the committed snapshots (e.g., `logout_token_native_auth` snapshot at line 878
    produces different HTTP request/response sequences). This is a pre-existing insta
    snapshot regression that appears in both auth and secret_service tests.

  (2) LLM only: Python 3.12 not installed in Docker
    The uv integration test suite at `crates/uv-test/src/lib.rs:2087` contains a
    `find_python` helper that locates Python interpreters of specific versions to
    create test virtual environments. The test setup panics with:
      "Could not find Python 3.12 for test. Try `cargo run python install` first"
    The uv test harness requires a Python 3.12 installation accessible at a standard
    path or via the uv managed toolchain. The LLM's Dockerfile doesn't install Python
    3.12 or run `cargo run python install` to set up the managed Python toolchain.
    All 387 LLM failures share this root cause: the tests panic immediately at setup
    before any actual test code executes.

  (3) Test scope difference: original=42 (filtered), LLM=2654 (full workspace)
    The original CI uses `--profile ci-linux` which applies a custom nextest profile
    that limits the test scope with a setup hook script (`scripts/nextest-setup-hook-unix.sh`)
    and runs only a filtered subset of tests that need specific external credentials.
    The LLM runs the full workspace without the profile filter, discovering 2654 tests.
