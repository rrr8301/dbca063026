  [Root cause classification]
  Three different failures with distinct root causes:

  (1) Pre-existing: options::value_requires_long_or_short (both environments)
    The `just` error reporting for the `[arg(..., value='baz')]` attribute validation
    points to the wrong column in the error message. The error says:
      `error: Argument attribute 'value' only valid with 'long' or 'short'`
    The actual error highlights column 2 (the `[` bracket) but the test expects column 13
    (the `value` keyword position). This is a pre-existing bug in just's error location
    reporting for argument attribute validation — the span points to the start of the
    attribute syntax instead of the specific `value=` identifier.

  (2) Missing shell for backtick execution (LLM Docker only)
    The `backticks::trailing_newlines_are_stripped` test uses a justfile with:
      `foos := \`print('foo' * 4)\``
    The justfile specifies a shell like `set shell := ["python3", "-c"]` to interpret
    backtick expressions using Python (since `print('foo' * 4)` is Python syntax producing
    "foofoofoofoo"). In the Docker image, the specified shell interpreter is not installed
    or not in the expected PATH, causing just to fail with:
      `Backtick could not be run because just could not find the shell: No such file or directory`
    On GitHub Actions ubuntu-latest, the required interpreter is available and `foos` gets
    the value "foofoofoofoo" correctly.

  (3) Environment variable not set in Docker (LLM Docker only)
    `functions::env_var_functions` tests just's `env_var()` and `env_var_or_default()`
    functions by reading actual environment variables. At `tests/functions.rs:86`, the test
    calls `.unwrap()` on `std::env::var("SOME_VAR")` where the expected variable is not
    set in the Docker container:
      `called Result::unwrap() on an Err value: NotPresent`
    GitHub Actions automatically sets environment variables (e.g., `RUNNER_OS`,
    `GITHUB_WORKSPACE`, `HOME`, or other CI-specific vars) that this test relies on.
    The Docker container's minimal environment doesn't include those variables.
