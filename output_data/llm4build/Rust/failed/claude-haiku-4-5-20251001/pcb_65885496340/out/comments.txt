  [Root cause classification]
  Two distinct failure causes produce 195 LLM failures vs 6 original:

  (1) Pre-existing: 6 insta snapshot failures (both original CI and LLM)
    The `pcb-zen-core::input` test group tests config conversion, I/O, and type inference
    using the `insta` crate for snapshot comparisons. Six tests fail because committed
    snapshots at `crates/pcb-zen-core/tests/snapshots/` no longer match current output:
    - config_convert_preserves_correct_types
    - config_convert_with_enum
    - config_convert_chain
    - config_convert_with_default
    - config_with_convert_function
    - io_config_with_help_text
    These represent a pre-existing snapshot regression where the implementation changed
    but the reference snapshots weren't updated.

  (2) git sparse checkout of diodeinc/stdlib fails in Docker (LLM only — 189 failures)
    The pcb project's integration tests in `pcb-test-utils/src/sandbox.rs` set up test
    sandbox environments that fetch the `stdlib` library from `github.com/diodeinc/stdlib`
    via `git sparse-checkout`. The test infrastructure at `sandbox.rs:665` calls `.unwrap()`
    on the fetch result, which panics when it fails with:
      `Failed to fetch github.com/diodeinc/stdlib via git sparse checkout: No such file or directory (os error 2)`

    The `No such file or directory (os error 2)` error indicates that `git` binary itself
    cannot be found in PATH when the test subprocess tries to invoke `git sparse-checkout`.
    The Docker container either doesn't have git installed, has it at a non-standard PATH
    that isn't inherited by the test subprocess, or the test environment's sandbox
    isolation removes PATH entirely.

    On GitHub Actions, git is pre-installed system-wide and available in PATH for all
    subprocesses. All 189 tests that use the git sandbox to fetch `diodeinc/stdlib` pass
    on the original CI.

    The impacted test modules include: pcb::auto_deps, pcb::bom, pcb::build,
    pcb::electrical_checks, pcb::info, pcb::moved, pcb::netlist, pcb::path, pcb::release,
    pcb::simple, pcb::tag, pcb::test_bench, pcb-layout, pcb-test-utils, pcb-zen, and
    pcb-zen-core — essentially every test that spawns a git-based sandbox environment.
