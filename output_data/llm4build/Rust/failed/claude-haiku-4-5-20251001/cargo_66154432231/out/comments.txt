  [Root cause classification]
  Two separate root causes produce 94 new failures in the LLM Docker:

  (1) `COPY . .` in Dockerfile doesn't preserve `in` symlinks → EISDIR on 93 tests
    The cargo test suite (using `snapbox`/`cargo-test-support`) stores per-test fixture
    directories under `tests/testsuite/<command>/<test_name>/in/` and `out/`. These `in`
    paths in the git repository are SYMLINKS pointing to another directory (e.g., the actual
    fixture data is stored elsewhere). When Docker's `COPY . .` instruction copies the
    repository, it follows symlinks and materializes their targets as real directories rather
    than preserving them as symlinks.

    At test runtime, `crates/cargo-test-support/src/lib.rs:373` calls `fs::copy(src, dst)`
    on the `in` path, expecting a single-file copy (treating `in` as a symlink to a file).
    But in the Docker container, `in` is now a real directory. `std::fs::copy()` cannot copy
    directories — it returns `EISDIR (os error 21)`. This panics the test with:
      `Failed to copy .../in to .../case/case/: Is a directory (os error 21)`

    All 93 tests that use these symlinked fixture directories fail in the Docker container.
    In the original CI (aarch64 GitHub Actions runner), git clone preserves symlinks correctly
    so `in` remains a symlink and `fs::copy` succeeds.

  (2) i686 cross-compilation target not installed → artifact_dep cross test fails
    `artifact_dep::cross_doctests_works_with_artifacts` attempts to cross-compile to
    `i686-unknown-linux-gnu`. On the original aarch64 CI runner, the i686 cross-compile
    toolchain may be available. In the LLM's x86_64 Docker container, the i686 target is
    not installed (`rustup target add i686-unknown-linux-gnu` is needed).

  (3) Architecture difference: original CI is aarch64, LLM Docker is x86_64
    The original CI runs on Linux aarch64 (ARM64). The LLM Docker runs on x86_64. Some
    tests may produce different behavior on different architectures (endianness, pointer
    size, etc.), accounting for the 3 additional tests collected in LLM.
