  [Root cause classification]
  Two separate environment mismatches cause two different test failures:

  (1) Original CI — missing snapshot file for text_edit_halign (macOS)
    egui uses image snapshot tests: tests render a widget and compare the output against a
    reference PNG stored in `tests/egui_tests/tests/snapshots/`. The snapshot file
    `text_edit_halign.png` was never committed to the repository or was recently deleted.
    The snapshot testing framework (egui_kittest) panics when the expected PNG is missing:
      `Missing snapshot: .../snapshots/text_edit_halign.png.
       Run UPDATE_SNAPSHOTS=1 cargo test --all-features to update the snapshots.`
    The snapshot file must first be generated with `UPDATE_SNAPSHOTS=1` and then committed.
    This test therefore fails as a pre-existing issue in the macOS CI.

  (2) LLM Docker — wgpu GPU adapter not found (Linux headless)
    The `test_demo_app` integration test in `crates/egui_demo_app/` uses egui's wgpu backend
    (`egui_kittest/src/wgpu.rs`) to render the egui demo application and capture output for
    verification. The test calls `wgpu::Instance::create_adapter(...)` to initialize a GPU
    render pipeline. In a headless Linux Docker container with no physical or virtual GPU,
    wgpu finds no suitable Vulkan/OpenGL adapter and panics:
      `Failed to create render state: CustomNativeAdapterSelectionError("No adapter found")`
    On macOS ARM64 CI, the runner has Metal GPU support available so `test_demo_app` succeeds;
    the snapshot-based `text_edit_halign` test is the one that fails instead.

  (3) Test count difference: 97 (original) vs 45 (LLM)
    The 52-test gap arises from platform differences:
    - macOS has native Metal GPU support enabling more wgpu/GPU-backed tests to build and run
    - On headless Linux Docker, crates with GPU/display backend requirements (egui_kittest
      wgpu tests, windowing tests) may be skipped or compile differently under the
      `--all-features` flag, reducing the set of runnable tests
    - The original macOS runner includes tests from `tests/egui_tests/`, `tests/test_viewports/`,
      and other workspace members that may not build on Linux without display server
