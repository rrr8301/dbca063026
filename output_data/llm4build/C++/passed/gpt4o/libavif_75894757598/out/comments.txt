  [Root cause classification]
  - The LLM Dockerfile installs Rust for the root user (`/root/.cargo`) and then switches to `testuser`, but run.sh sets `PATH="/home/testuser/.cargo/bin:/root/.cargo/bin:${PATH}"`. At CMake configure time, corrosion (the Rust/CMake bridge used to build rav1e) uses `find_package(Rust)` which looks in PATH and `~/.cargo/bin` for `rustc`. Despite PATH being set, the Rust binaries are owned by root (`/root/.cargo/bin`) and the non-root user cannot execute them correctly in all environments.
  - After 20 attempts, the LLM could not resolve the cross-user Rust toolchain visibility issue, resulting in TR=False with no CTest output on any attempt.
  - In the original CI the runner installs Rust under the default user and cargo/rustc are directly accessible in PATH without cross-user ownership complications.
