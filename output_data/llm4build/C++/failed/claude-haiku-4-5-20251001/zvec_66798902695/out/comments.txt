        count and scheduler, which differ between the original runner and the Docker host.
      * schema_test FAILS on the original runner (schema_test.cc:830..934) but does not fail
        in the container.
    The build auto-detected AVX-512 and selected -march=graniterapids, enabling the AVX-512

    environment-independent failure of that SIMD code path). Net effect: run count (128) and
    failure count (6) match exactly, but because one CPU/threading-sensitive test flips in
    each direction (thread_queue_test in, schema_test out), the failing-name set does not
    match and the run is not fully reproducible.
